## Account Management

<ExpandableGroup>
  <Expandable title="How do I create an Esen account?" default-open="true">

  Visit `https://esen-docs.com/signup` and provide your email, name, and password. Verify your email to activate the account. You gain immediate access to your default documentation space.

  </Expandable>

  <Expandable title="What if I forget my password?">

  Click "Forgot Password" on the login page. Enter your email, and follow the reset link sent to your inbox. Reset links expire after `{24}` hours for security.

  </Expandable>

  <Expandable title="Can I use single sign-on (SSO)?">

  Enterprise plans support SSO via SAML or OAuth. Contact support at `support@esen-docs.com` to enable it for your organization.

  </Expandable>
</ExpandableGroup>

## Best Practices

Follow these guidelines to maintain high-quality documentation in Esen.

<Callout kind="success" default-open="true">
  Use version control integration for collaborative editing.
</Callout>

### Organize Content Effectively

- Group pages into spaces for different projects.
- Use frontmatter for metadata like `title` and `description`.

### Integrate with Your Workflow

<CodeGroup tabs="npm,yarn">
  ```bash
  npm install esen-docs-sdk
  ```
  ```bash
  yarn add esen-docs-sdk
  ```
</CodeGroup>

```javascript
// Example: Publish a doc programmatically
import { EsenClient } from 'esen-docs-sdk';

const client = new EsenClient({ apiKey: 'YOUR_API_KEY' });
await client.docs.publish({ spaceId: 'proj-123', content: markdown });
```

### Advanced Tips

<Expandable title="How to migrate from other platforms?">
  Export Markdown from your current tool, then bulk import via Esen's API.

  ```bash
  curl -X POST https://api.esen-docs.com/v1/import \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -F "file=@docs.zip"
  ```
</Expandable>
