## Overview

Get your Esen documentation space running in under 5 minutes. Follow these steps to create an account, set up a new space, and upload your initial content. Esen handles the rest, providing a clean, searchable docs site for your project.

<Callout kind="tip">
  New to Esen? No prior setup required. Use your GitHub account for instant access.
</Callout>

## Step-by-Step Setup

Use these sequential steps to launch your first space.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Visit [esen-docs.com](https://esen-docs.com) and click **Sign Up**.

    Choose GitHub, Google, or email signup. Verify your email if prompted.

    <Image
      src="https://esen-docs.com/signup-screenshot.png"
      alt="Esen signup screen showing GitHub and Google options"
      width="800"
      height="500"
    />
  </Step>

  <Step title="Create New Space" icon="plus">
    From the dashboard, click **New Space**.

    Enter a name like `my-project-docs` and select a template (e.g., "API Docs" or "Blank").

    Confirm and navigate to your new space at `/spaces/my-project-docs`.
  </Step>

  <Step title="Upload Initial Content" icon="upload">
    In your space, create your first page via **New Page**.

    Choose Markdown or MDX format. Paste or upload your docs.

    Preview and publish. Your site updates live.
  </Step>
</Steps>

## Upload Methods

Choose your preferred method to add content.

<Tabs>
  <Tab title="Web UI" icon="globe">
    Drag-and-drop files directly into the editor.

    Supports `.md`, `.mdx`, images, and more.
  </Tab>

  <Tab title="GitHub Sync" icon="github">
    Connect your repo for automatic sync.

    ```bash
    # Add Esen as remote
    git remote add esen https://git.esen-docs.com/my-project-docs.git
    git push esen main
    ```
  </Tab>

  <Tab title="CLI" icon="terminal">
    Install and use the Esen CLI.

    <CodeGroup tabs="npm,yarn">
      ```bash
      npm install -g @esen/cli
      ```
      ```bash
      yarn global add @esen/cli
      ```
    </CodeGroup>

    ```bash
      esen init my-project-docs
      esen upload --space my-project-docs
    ```
  </Tab>
</Tabs>

## Advanced Configuration

<Expandable title="Optional: API Integration" default-open="false">

Set up programmatic uploads with the Esen API.

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X POST https://api.esen-docs.com/v1/spaces/my-project-docs/pages \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "path": "quickstart.mdx",
      "content": "# Hello World"
    }'
  ```
  ```javascript
  const response = await fetch('https://api.esen-docs.com/v1/spaces/my-project-docs/pages', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      path: 'quickstart.mdx',
      content: '# Hello World',
    }),
  });
  ```
</Request>

</Expandable>
